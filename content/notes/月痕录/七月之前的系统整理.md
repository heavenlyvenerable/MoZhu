# 七月之前的系统整理

六月的工作有一种反复试探的感觉。页面风格换过几次，参考网站也换过几次，从古风场景、卷轴背景、印章入口，到更现代的博客式布局，再到黑色后台和本地内容管理。每一次修改表面上像是审美变化，实际上是在逐渐明确这个网站到底要解决什么问题。

现在看来，它不是传统博客，也不是纯展示页，而是一个带个人审美的本地工作台。前台可以有漂亮的过渡和阅读体验，但后台必须足够务实：分类、文件、属性、导入、索引都要可控。只有后台数据真实，前台的所有数字、列表和动画才不会失去意义。

六月暴露出的一个问题是，过早追求效果会让基础结构变得混乱。比如页面上有很多示例文章时，看起来很丰富，但真正要导入自己的文件时，反而不知道哪些是真内容，哪些只是占位。七月开始应该反过来，先让内容链路可靠，再逐步恢复视觉细节。

下个月的重点应该放在“少而准”。每个分类先有少量真实 Markdown 文件，每个文件都有明确属性，每个页面都只展示来自文件系统或索引的数据。等这个底座稳定，再扩展照片、视频、日记、知识库等更复杂的内容类型。
